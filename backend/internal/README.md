# /internal

このディレクトリには、アプリケーションの核となるビジネスロジックと、その実行に必要な実装を配置します。
Goの `internal` パッケージの制約により、本プロジェクト外からのインポートは禁止されており、カプセル化が保証されています。

## 各パッケージの詳細

### 📂 domain
システムの「ルール」と「データ構造」を定義します。

### 📂 usecase
ユーザーの「やりたいこと（機能）」を記述します。
- ドメインオブジェクトを操作して、特定のビジネスフローを完結させます。
- 外部への副作用（DB保存など）は、`domain` で定義されたインターフェースを介して行います。

### 📂 interface
外部からの入力を `usecase` が理解できる形式に変換し、結果を外部に返却します。

### 📂 infrastructure
技術的な詳細（外部ライブラリやミドルウェアの操作）をカプセル化します。

# Backend Project Overview

このリポジトリは、Go言語で構築されたバックエンドシステムのソースコードを管理しています。

## 1. プロジェクトの全体像
個人的好みで、クリーンアーキテクチャに似たレイヤー分割を行っています。



---

## 2. ディレクトリ構造
主要なディレクトリの役割は以下の通りです。

| ディレクトリ | 内容・役割 |
| :--- | :--- |
| `cmd/` | **エントリポイント**。メイン関数（`main.go`）を配置します。サーバー起動やCLIツールなど、実行バイナリごとにサブディレクトリを作成します。 |
| `internal/` | **非公開コード**。このプロジェクト内でのみ利用されるロジックです。外部プロジェクトからインポートされるのを防ぎ、カプセル化を保ちます。 |
| `deployments/` | **デプロイ設定**。Docker、Kubernetes、Terraform、CI/CDの設定ファイルなど、運用に関する構成を管理します。 |
| `go.mod` / `go.sum` | **依存関係管理**。プロジェクトで使用する外部ライブラリのバージョンを管理します。 |

---

## 3. 基本設計方針（アーキテクチャ）
`internal/` 配下では、以下のレイヤー構造を意識した実装を推奨します。

1.  **Handler (Interface Adapter)**: HTTPリクエストの解析やレスポンスの整形。
2.  **Usecase (Application Business Logic)**: アプリケーション固有のビジネスルール。
3.  **Repository (Data Access)**: データベースや外部ストレージとのやり取り。
4.  **Domain (Enterprise Business Logic)**: エンティティ（構造体）やインターフェースの定義。
